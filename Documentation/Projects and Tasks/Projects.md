# Projects

Projects organize the work in Relay. Each project is shown by its full name, with tags and tools for filtering and allocation context. Relay still keeps an internal code for compatibility, but it is not used as the visible project identity.

## Project fields

- Name: the human-readable project title.
- Internal code: generated from the project name for compatibility and not shown in project labels.
- Tags: `print`, `cg`, and `ai`.
- Tools: Houdini, Comfy, Nanobanana, Blender, and Unreal.

## Creating projects

Admin and Manager users can create projects from the Projects view. Artist and Client users can inspect projects they can access, but creation controls are disabled.

## Creating tasks

When a project is selected, Admin and Manager users can create project tasks with title, assignee, phase, priority, due date, and client visibility. New tasks appear in the task workflow immediately for the current session.

## Archiving projects

Project archive is restoreable. Archiving a project moves the project, its active tasks, and related allocations into Archive. Restoring the project returns those records to the active app views.
