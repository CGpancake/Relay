# Projects

Projects group deliverables, allocation records, tags, tools, and studio ownership.

## Project register

Projects are grouped by studio in collapsible sections. Selecting a project opens its detail panel with editable studio, tag, and tool metadata for Admin and Manager users.

## Creating projects

Admin and Manager users can create projects with a name, studio, tags, and tools. The app creates an internal code and local id for the current session. Artist and Client users can inspect project data but creation controls are disabled.

## Creating deliverables

When a project is selected, Admin and Manager users can create project deliverables with title, assignee, phase, priority, due date, and client visibility. New deliverables appear in Deliverables immediately for the current session.

## Archive behavior

Project archive is restoreable. Archiving a project moves the project, its active deliverables, and related allocations into Archive. Restoring the project returns those records to the active app views.
